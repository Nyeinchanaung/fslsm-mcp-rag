## Goal

Take the reasoning plan from Phase 1 and a raw student question, then produce a **reformulated query** optimized for FSLSM-aligned chunk retrieval from the FAISS vector store. This agent implements the core R0 vs R1 retrieval difference — the mechanism tested by CR@5 and Essential Recall metrics.

## R0 vs R1 Retrieval Pipeline (Conceptual)

```
R0 (Generic / Control):
  question ──> FAISS embed ──> top-5 by cosine similarity ──> chunks

R1 (Personalized / Experimental):
  question ──> ProfileAgent ──> reasoning_plan
                                    |
               reformulate_query <──┘
                    |
                    v
              FAISS embed ──> top-5 by cosine ──> FSLSM rerank ──> chunks
```

### Task 2.3 — FSLSM-Aware Reranking

This is the **core personalization mechanism** for retrieval — tested by CR@5 and Essential Recall.

```
[ ] 2.3.1  Implement method: rerank_chunks(chunks: List[dict], reasoning_plan: dict) -> List[dict]
[ ] 2.3.2  Boost chunks whose metadata tags overlap with reasoning_plan["reranking_bias"]
           - Score adjustment: +0.05 per matching tag
[ ] 2.3.3  Demote chunks tagged with reasoning_plan["deprioritize"]
           - Score adjustment: -0.05 per matching tag
[ ] 2.3.4  Re-sort chunks by adjusted score (descending)
[ ] 2.3.5  Note: chunk metadata tags need to be derived from chunk content
           - Option A: Pre-tag chunks with content type labels (visual_content, step_by_step, etc.)
           - Option B: Use keyword/heuristic matching at rerank time
           - Decision: Document which approach is chosen and why
```

**Reranking Tag Schema:**
```python
CONTENT_TAGS = [
    "visual_content",        # diagrams, charts, figures mentioned
    "step_by_step",          # numbered steps, sequential procedures
    "concrete_examples",     # specific numbers, worked examples
    "conceptual_overview",   # big-picture summaries, introductions
    "abstract_theory",       # theoretical frameworks, proofs
    "exercises",             # practice problems, exercises
    "definitions",           # formal definitions, terminology
    "analogies",             # metaphors, real-world comparisons
]
```

## Key Design Decision: Chunk Content Tagging

The reranking mechanism requires chunks to have content-type tags. Two approaches:

**Option A: Pre-tagging (Recommended)**
- Run a one-time script to tag all chunks in the FAISS index with content types
- Store tags in a sidecar JSON: `data/processed/chunks/chunk_tags.json`
- Pro: Fast at query time, consistent
- Con: Requires upfront effort

**Option B: Runtime heuristic**
- Use keyword matching at rerank time (e.g., "diagram" in text -> "visual_content")
- Pro: No preprocessing needed
- Con: Less accurate, slower

Document the chosen approach in the code docstring.
