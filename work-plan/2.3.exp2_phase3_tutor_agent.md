## Goal

---

### Task 3.2 — Implement `run_session()` Pipeline

The 6-step pipeline that executes for each session:

```
[ ] 3.2.1  Step 1 — Profile Resolution
           If mode == "R1":
             plan = profile_agent.generate_reasoning_plan(fslsm_vector)
             system_prompt = profile_agent.generate_system_prompt(plan)
           Else (R0):
             plan = None
             system_prompt = "You are a helpful tutor."

[ ] 3.2.2  Step 2 — Retrieval
           result = retrieval_agent.retrieve(
               question, reasoning_plan=plan,
               personalized=(mode == "R1")
           )

[ ] 3.2.3  Step 3 — Build LLM Context
           Assemble the full prompt:
           - System prompt (from Step 1)
           - Retrieved chunks as numbered evidence blocks (from Step 2)
           - Student question
           Implement: _build_context_prompt(chunks, question) -> str

[ ] 3.2.4  Step 4 — Call Tutor LLM
           Call GPT-4.1 mini via llm_client.chat()
           Record: response text, latency_ms, token_count

[ ] 3.2.5  Step 5 — Get Engagement Score
           Call virtual student agent to rate the response (1-5)
           Implement: _get_engagement_score(response, fslsm_vector) -> int

[ ] 3.2.6  Step 6 — Return session_result dict
           Assemble all fields into the output contract
```

---

### Task 3.3 — Context Prompt Builder

```
[ ] 3.3.1  Implement: _build_context_prompt(chunks: list, question: str) -> str
[ ] 3.3.2  Format retrieved chunks as numbered evidence blocks:
           """
           Use the following evidence to answer the student's question.

           [Evidence 1]: {chunk_text_1}
           [Evidence 2]: {chunk_text_2}
           ...
           [Evidence 5]: {chunk_text_5}

           Student Question: {question}
           """
[ ] 3.3.3  Respect max context window:
           - R0: <= 4000 tokens for context
           - R1: <= 2000 tokens for context (leaves room for richer system prompt)
           - Truncate chunks if total exceeds limit
```

---

### Task 3.4 — Engagement Score from Virtual Student Agent

```
[ ] 3.4.1  Implement: _get_engagement_score(response: str, fslsm_vector: dict) -> int
[ ] 3.4.2  Build a student persona prompt using behavioral_instructions from
           data/fslsm/profiles.json matching the fslsm_vector
[ ] 3.4.3  Call student agent (GPT-4.1 mini) with:
           - System prompt: student persona (FSLSM behavioral instructions)
           - User prompt: "A tutor gave you this response: '{response}'.
             Rate how engaging and well-suited this response is for your
             learning style on a scale of 1-5.
             1=completely mismatched, 5=perfectly aligned.
             Respond with ONLY a single integer."
[ ] 3.4.4  Parse response to extract integer 1-5
           - Handle parsing failures: default to 3, log warning
[ ] 3.4.5  For R0 baseline: still collect engagement using the same student persona
           (this is key — same student judges both R0 and R1 responses)
```

## LLM Cost Notes

Per the overall plan, the LLM assignments for Experiment 2:

| Role | Model | Cost |
|---|---|---|
| Tutor Agent (R0 & R1) | GPT-4.1 mini | $0.40/M input, $1.60/M output |
| Student Agent (engagement) | GPT-4.1 mini | Same |
| LLM-as-a-Judge (RR) | GPT-4o | $2.50/M input, $10/M output |

The judge model (GPT-4o) is used only in Phase 5 evaluation, not during session generation.
