## Goal

Orchestrate 4,000 interaction sessions (80 agents x ~50 questions x 2 conditions R0/R1), log all results to JSONL files, handle resumption after interruption, and report progress. This is the main experiment execution script.

## Tasks

### Task 4.1 — Load and Validate Inputs

```
[ ] 4.1.1  Load validated_agents.json — assert exactly 80 agents
           - Each agent has: agent_id, fslsm_vector, profile_label
[ ] 4.1.2  Load d2l_qa_groundtruth.json (or d2l_multihop_ground_truth.json)
[ ] 4.1.3  Stratified sample ~50 questions per agent:
           - Balance Gold / Silver quality tiers (if available)
           - Balance question types (synthesize_workflow, compare, explain, etc.)
           - Reproducible: random.seed(42)
[ ] 4.1.4  Save sampled question assignments to data/exp2/sampled_questions.json
           - Run this sampling ONCE, then reuse on subsequent runs
           - Format: list of {agent_id, question_id, question, gold_chunk_ids, ...}
[ ] 4.1.5  Create data/exp2/ directory if it doesn't exist
```

**Note:** The ground truth currently has ~30 multi-hop questions (9 confirmed in JSON). The overall plan targets ~50 per agent. If fewer questions are available, adjust the plan: either use all available questions or generate additional ones. Document the actual count.

---

### Task 4.2 — Build Session List

```
[ ] 4.2.1  For each (agent, question) pair: create TWO session dicts
           - R0 session: mode="R0"
           - R1 session: mode="R1"
[ ] 4.2.2  Total sessions = agents x questions x 2 conditions
           - Target: 80 x 50 x 2 = 8,000 (adjust if fewer questions)
[ ] 4.2.3  Include gold_chunk_ids in each session dict (for CR@5 / ER evaluation in Phase 5)
[ ] 4.2.4  Shuffle session list with seed=42 to avoid systematic API rate patterns
```

**Session dict format:**
```python
{
    "agent_id": "agent_007",
    "profile_label": "Active-Sensing-Visual-Global",
    "fslsm_vector": {"act_ref": -1, "sen_int": -1, "vis_ver": -1, "seq_glo": 1},
    "question_id": "d2l_mh_0042",
    "question": "How does backpropagation work?",
    "gold_chunk_ids": ["abc_0", "xyz_0", "ghi_0", "pqr_0", "stu_0"],
    "mode": "R1"
}
```

### Task 4.3 — Async/Batch Runner

```
[ ] 4.3.1  Implement the main run loop using asyncio or ThreadPoolExecutor
           - max_workers=5 (to stay within API rate limits)
[ ] 4.3.2  For each session: call tutor_agent.run_session(session)
[ ] 4.3.3  Stream-write results to JSONL files (one line per session):
           - R0 results -> experiments/exp2_tutor_personalization/results/raw_sessions_r0.jsonl
           - R1 results -> experiments/exp2_tutor_personalization/results/raw_sessions_r1.jsonl
[ ] 4.3.4  Display progress bar via tqdm
[ ] 4.3.5  Create results/ directory if it doesn't exist
```

---

### Task 4.4 — Checkpoint/Resume Logic

Critical for a ~$47 experiment that can be interrupted.

```
[ ] 4.4.1  Before each session, check if (agent_id, question_id, mode) already exists
           in the output JSONL file
[ ] 4.4.2  Skip already-completed sessions (allows safe re-run after interruption)
[ ] 4.4.3  On startup: report how many sessions are already completed vs remaining
[ ] 4.4.4  Implementation: load existing JSONL into a set of (agent_id, question_id, mode)
           tuples at startup for O(1) lookup
```

---

### Task 4.5 — Error Handling & Logging

```
[ ] 4.5.1  Catch all per-session exceptions (API errors, timeouts, parsing failures)
[ ] 4.5.2  Write error records to JSONL with traceback:
           {"agent_id": "...", "question_id": "...", "mode": "...", "error": "...", "traceback": "..."}
[ ] 4.5.3  Continue processing remaining sessions after individual failures
[ ] 4.5.4  At completion, report final counts:
           - Success / Error / Skipped (previously completed)
[ ] 4.5.5  Log to both console (via rich/tqdm) and a log file:
           experiments/exp2_tutor_personalization/results/run_exp2.log
```

---

### Task 4.6 — CLI Interface

```
[ ] 4.6.1  Implement argparse CLI:
           python run_exp2.py --mode R0          # run only baseline
           python run_exp2.py --mode R1          # run only personalized
           python run_exp2.py --mode both        # run full experiment (default)
           python run_exp2.py --dry-run --n 5    # test 5 sessions only
           python run_exp2.py --workers 3        # override concurrency
[ ] 4.6.2  --dry-run: run N sessions without writing to output files (for testing)
[ ] 4.6.3  --resume: explicitly enable resume mode (default: on)
```
