## 1. Experiment Overview

**Design:** A/B simulation study comparing two system configurations across 4,000 interaction sessions.

| Group | Config ID | Description |
|---|---|---|
| Control | **R0** | Generic RAG — retrieves by semantic similarity only; system prompt: *"You are a helpful tutor."* |
| Experimental | **R1** | Personalized RAG — retrieves and generates conditioned on the agent's FSLSM vector via PersonaRAG pipeline |

**Workload:**
- **80 validated agents** (16 binary FSLSM profiles × 5 instances) from Experiment 1
- **~50 questions per agent** drawn from the D2L-QA multi-hop 5-chunk ground truth dataset
- **4,000 total sessions** — for each query-agent pair, both R0 and R1 generate a response

**LLM Assignments:**

| Role | Model | Rationale |
|---|---|---|
| Tutor Agent (R0 & R1) | **GPT-4.1 mini** | PRA 0.994 in Exp 1; strong instruction following; $0.40/$1.60 per M tokens |
| Student Agent (engagement) | **GPT-4.1 mini** | Same model family; consistent persona injection; avoids cross-provider noise |
| LLM-as-a-Judge (RR) | **GPT-4o** | Frontier judgment quality required for reliable 1–5 RR scoring |

**Projected Budget (OpenAI Track):**

| Role | Model | Input | Output | Cost |
|---|---|---|---|---|
| Tutor Agent ×8,000 | GPT-4.1 mini ($0.40/M, $1.60/M) | 10.8M → $4.32 | 3.2M → $5.12 | **$9.44** |
| Student Agent ×8,000 | GPT-4.1 mini | 5.2M → $2.08 | 0.4M → $0.64 | **$2.72** |
| LLM-as-a-Judge ×8,000 | GPT-4o ($2.50/M, $10/M) | 6.4M → $16.00 | 0.8M → $8.00 | **$24.00** |
| Subtotal | | | | **$36.16** |
| +30% buffer (retries, dev runs) | | | | **~$47 USD** |



| Metric | Symbol | Target (R1 vs R0) |
|---|---|---|
| Style Conformance Score | SCS | ↑ significantly (0.45 → 0.70) |
| Response Relevance | RR | ≈ stable (4.1 → 4.2) |

## Phase 1 — Profile Agent

**File:** `src/tutor/profile_agent.py`  
**Role:** Translates a binary FSLSM vector `[-1|+1, -1|+1, -1|+1, -1|+1]` into a natural-language **reasoning plan** that downstream agents use to shape retrieval and generation.

### 1.1 Input / Output Contract

```python
# Input
fslsm_vector: dict = {
    "act_ref": -1,   # -1=Active, +1=Reflective
    "sen_int": -1,   # -1=Sensing, +1=Intuitive
    "vis_ver": -1,   # -1=Visual, +1=Verbal
    "seq_glo": +1    # -1=Sequential, +1=Global
}

# Output
reasoning_plan: dict = {
    "style_label": "Active-Sensing-Visual-Global",
    "retrieval_directive": "Retrieve concrete worked examples with diagrams; prioritize conceptual overviews before step-by-step details.",
    "generation_directive": "Begin with the big picture. Use visual scaffolding (diagrams, charts). Ground in concrete facts. Encourage hands-on thinking.",
    "reranking_bias": ["visual_content", "conceptual_overview", "concrete_examples"],
    "deprioritize": ["abstract_theory", "step_by_step_only"]
}
```

### 1.2 Implementation Tasks

```
[ ] 1.2.1  Define FSLSM_STYLE_MAP — a lookup table (16 profiles × 4 dimensions)
           mapping each binary combination to:
           - human-readable style_label
           - retrieval_directive (what kind of chunks to seek)
           - generation_directive (how to instruct the LLM tutor)
           - reranking_bias (list of content tags to boost)
           - deprioritize (list of content tags to demote)
           Source: Graf et al. (2007) dimension descriptors

[ ] 1.2.2  Implement class ProfileAgent with method:
           generate_reasoning_plan(fslsm_vector: dict) -> dict
           - Validate vector (4 dimensions, values ∈ {-1, +1})
           - Look up FSLSM_STYLE_MAP
           - Return reasoning_plan dict

[ ] 1.2.3  Add generate_system_prompt(reasoning_plan: dict) -> str
           - Returns the tutor system prompt string for injection into LLM call
           - Example: "You are an expert AI Tutor. The student is a Visual-Sensing-Global
             learner with a Reflective processing style. Do not proceed step-by-step
             without first establishing how each step connects to the whole..."

[ ] 1.2.4  Unit test: test_profile_agent.py
           - Test all 16 binary profiles produce valid, non-empty reasoning plans
           - Test that act_ref=-1 (Active) maps to retrieval_directive containing
             "interactive" or "hands-on"
           - Test that vis_ver=-1 (Visual) maps to generation_directive containing
             "diagram" or "visual"
```

### 1.3 Key Data Structure — `FSLSM_STYLE_MAP`

```python
FSLSM_STYLE_MAP = {
    # Format: (act_ref, sen_int, vis_ver, seq_glo)
    (-1, -1, -1, -1): {  # Active-Sensing-Visual-Sequential
        "retrieval_directive": "Retrieve step-by-step visual walkthroughs with concrete examples...",
        "generation_directive": "Use numbered diagrams and concrete procedural steps...",
        ...
    },
    (-1, -1, -1, +1): {  # Active-Sensing-Visual-Global
        ...
    },
    # ... all 16 combinations
}
```

### 1.4 Done Criteria
- `ProfileAgent.generate_reasoning_plan()` returns valid dict for all 16 FSLSM profiles
- System prompt is ≥ 3 sentences and references the learner's style explicitly
- All unit tests pass

## Phase 2 — Retrieval Agent

**File:** `src/tutor/retrieval_agent.py`  
**Role:** Takes the reasoning plan from Phase 1 and a raw student question, then produces a **reformulated query** optimised for FSLSM-aligned chunk retrieval from the FAISS vector store.

### 2.1 Input / Output Contract

```python
# Input
question: str = "How does backpropagation work?"
reasoning_plan: dict  # from Phase 1 ProfileAgent

# Output
retrieval_result: dict = {
    "reformulated_query": "Explain backpropagation with visual diagrams showing gradient flow...",
    "retrieved_chunks": [ChunkObject, ...],   # top-5 chunks
    "chunk_ids": ["abc123_0", "def456_0", ...],
    "reranked": True,
    "scores": [0.91, 0.88, 0.84, 0.80, 0.77]
}
```

### 2.2 Implementation Tasks

```
[ ] 2.2.1  Implement query reformulation
           - Method: reformulate_query(question: str, reasoning_plan: dict) -> str
           - Append retrieval_directive to the raw question to create a richer query
           - Example: "How does backpropagation work? [Retrieve with visual scaffolding,
             concrete numerical examples, and step-by-step diagrams]"

[ ] 2.2.2  Implement FAISS retrieval wrapper
           - Method: retrieve_chunks(query: str, k: int = 5) -> List[Chunk]
           - Load FAISS index from data/vectorstore/d2l_faiss_index/
           - Use sentence-transformers embedding (same model as Exp 1)
           - Return top-k chunks with chunk_id, text, section_id, score

[ ] 2.2.3  Implement FSLSM-aware reranking
           - Method: rerank_chunks(chunks: List[Chunk], reasoning_plan: dict) -> List[Chunk]
           - Boost chunks whose metadata tags overlap with reasoning_plan["reranking_bias"]
           - Demote chunks tagged with reasoning_plan["deprioritize"]
           - Reranking is lightweight: adjust cosine scores by ±0.05 per matching tag
           - This is the core mechanism tested by CR@5 and Essential Recall

[ ] 2.2.4  R0 vs R1 switch
           - Method: retrieve(question, reasoning_plan=None, personalized=True)
           - If personalized=False (R0): skip reformulation and reranking; raw FAISS only
           - If personalized=True (R1): apply 2.2.1 → 2.2.2 → 2.2.3 pipeline

[ ] 2.2.5  Unit test: test_retrieval_agent.py
           - Test that R1 retrieval (personalized=True) returns different chunk ordering
             than R0 for a Visual-Global learner on a conceptual question
           - Test that CR@5 is computable from returned chunk_ids vs gold_chunk_ids
           - Test FAISS index loads without error
```

### 2.3 R0 vs R1 Retrieval Difference (Conceptual)

```
R0 (Generic):
  question ──► FAISS embed ──► top-5 by cosine similarity ──► chunks

R1 (Personalized):
  question ──► ProfileAgent ──► reasoning_plan
                                    │
               reformulate_query ◄──┘
                    │
                    ▼
              FAISS embed ──► top-5 by cosine ──► FSLSM rerank ──► chunks
```

### 2.4 Done Criteria
- `retrieve(personalized=False)` replicates standard FAISS retrieval (R0 baseline)
- `retrieve(personalized=True)` applies reformulation + reranking (R1)
- Both return the same schema (`chunk_ids`, `retrieved_chunks`, `scores`)

## Phase 3 — Tutor Agent

**File:** `src/tutor/tutor_agent.py`  
**Role:** LLM Orchestrator that wires together `ProfileAgent` + `RetrievalAgent` + LLM call to produce the final tutor response. This is the core of the PersonaRAG pipeline.

### 3.1 Input / Output Contract

```python
# Input
session: dict = {
    "agent_id": "agent_007",
    "fslsm_vector": {"act_ref": -1, "sen_int": -1, "vis_ver": -1, "seq_glo": +1},
    "question": "How does backpropagation work?",
    "question_id": "d2l_mh_0042",
    "mode": "R1"   # "R0" or "R1"
}

# Output
session_result: dict = {
    "agent_id": "agent_007",
    "question_id": "d2l_mh_0042",
    "mode": "R1",
    "response": "Let's begin with the big picture...",
    "system_prompt_used": "You are an expert AI Tutor...",
    "retrieved_chunk_ids": ["abc_0", "def_0", "ghi_0", "jkl_0", "mno_0"],
    "reformulated_query": "How does backpropagation work? [with visual diagrams...]",
    "engagement_score": 4,          # from virtual student agent (1-5)
    "latency_ms": 1240,
    "token_count": 312
}
```

### 3.2 Implementation Tasks

```
[ ] 3.2.1  Implement class TutorAgent with:
           - __init__(llm_client, profile_agent, retrieval_agent, mode="R1")
           - run_session(session: dict) -> dict

[ ] 3.2.2  Implement run_session() pipeline:
           Step 1: If mode == "R1":
                     plan = profile_agent.generate_reasoning_plan(fslsm_vector)
                     system_prompt = profile_agent.generate_system_prompt(plan)
                   Else (R0):
                     plan = None
                     system_prompt = "You are a helpful tutor."

           Step 2: result = retrieval_agent.retrieve(
                       question, reasoning_plan=plan,
                       personalized=(mode=="R1")
                   )

           Step 3: Build LLM context:
                   - System prompt (from Step 1)
                   - Retrieved chunks as context (from Step 2)
                   - Student question

           Step 4: Call LLM API → get response text

           Step 5: Request engagement score from the virtual student agent
                   (inject the response into the student agent persona prompt,
                    ask it to rate engagement 1–5)

           Step 6: Return session_result dict

[ ] 3.2.3  Implement _build_context_prompt(chunks, question) -> str
           - Formats retrieved chunks as numbered evidence blocks
           - Respects max context window (≤ 4000 tokens for R0, ≤ 2000 for R1)

[ ] 3.2.4  Implement _get_engagement_score(response, fslsm_vector) -> int
           - Calls student agent with its persona + the tutor response
           - Prompts: "Rate how engaging this response is for your learning style (1-5)"
           - Parse and return integer score

[ ] 3.2.5  Add retry logic + timeout handling for LLM calls
           - Max 3 retries with exponential backoff
           - Log failed sessions for re-run

[ ] 3.2.6  Integration test: test_tutor_agent.py
           - Smoke test: run one R0 session and one R1 session end-to-end
           - Assert session_result contains all required keys
           - Assert engagement_score ∈ {1, 2, 3, 4, 5}
           - Assert retrieved_chunk_ids has exactly 5 entries
```

## Phase 5 — Evaluation & Results

**File:** `experiments/exp2_tutor_personalization/evaluate_exp2.py`  
**Role:** Loads the JSONL logs, computes all metrics, runs statistical tests, and writes `exp2_results_summary.json`.

### 5.1 Implementation Tasks

```
[ ] 5.1.1  Compute SCS for every session
           - Load FSLSM style description embeddings from data/exp2/scs_style_anchors.json
             (16 entries, one per profile — e.g., "Use graphs, charts, flow diagrams")
           - For each session: SCS = cosine_similarity(embed(response), embed(style_anchor))
           - Use: src/evaluation/metrics.py → compute_scs(response, style_anchor)
           - Expected: R1 mean SCS ≈ 0.70, R0 mean SCS ≈ 0.45

[ ] 5.1.2  Compute RR for every session (LLM-as-a-Judge)
           - Use judge prompt from prompts/judge_prompts.py

           - Expected: R0 ≈ 4.1, R1 ≈ 4.2 (stable)

[ ] 5.1.3  Compute CR@5 and Essential Recall for every session
           - CR@5 = |retrieved_chunk_ids ∩ gold_chunk_ids| / |gold_chunk_ids|
           - Essential Recall = |retrieved_chunk_ids ∩ essential_chunk_ids| / |essential_chunk_ids|

[ ] 5.1.4  Use Engagement Score from session log (already collected during Phase 4)
```

## 10. Metric Definitions & Formulas

### Style Conformance Score (SCS)
Measures how well the tutor response aligns with FSLSM pedagogical guidelines for the target profile.

```
SCS(r, s) = (Vr · Vs) / (||Vr|| × ||Vs||)

Where:
  Vr = embedding of the tutor's text response
  Vs = embedding of the FSLSM style description anchor (Graf et al., 2007)
```

### Response Relevance (RR)
LLM-as-a-Judge scoring against the D2L ground truth answer.

```
RR = (1/N) × Σ Judge(rᵢ, gᵢ)

Where:
  Judge(*) → score ∈ {1, 2, 3, 4, 5}  (1=Irrelevant, 5=Highly Relevant)
  rᵢ = system response for session i
  gᵢ = gold answer for question i
```

### Chunk Recall@5 (CR@5)
```
CR@5 = |R ∩ G| / |G|

Where:
  R = set of top-5 retrieved chunk IDs
  G = set of gold (ground-truth) chunk IDs
```

### Essential Recall (ER)
```
ER = |R ∩ E| / |E|

Where:
  E = set of essential chunks (annotated in D2L-QA as must-have for correct answer)
```

### Engagement Score (Eng)
Self-reported by the virtual student agent (1–5), collected in Phase 4.
```
Engagement = Student agent's rating of tutor response relevance to its learning style
             (1 = completely mismatched, 5 = perfectly aligned)
```
