## Context: Why RR is inverted

Per-profile A/B analysis (Fig 8) shows that R0 outperforms R1 on Response Relevance
across all 16 FSLSM profiles. The inversion is largest for Visual + Sequential + Active
profiles. Root cause: style formatting directives (numbered steps, ASCII diagrams,
practice prompts) consume the 1,000-token output budget, leaving insufficient space
for complete factual coverage that the LLM judge measures.

Three fixes address this at three different layers:
- **Fix A** — Token budget layer (`tutor_agent.py`)
- **Fix B** — Generation directive framing (`profile_agent.py` — `_DIM_GENERATION`)
- **Fix C** — Factual anchor injection (`profile_agent.py` — `generate_system_prompt`)

## Fix A — Profile-aware token budget in `tutor_agent.py`

**File:** `src/tutor/tutor_agent.py`  
**Problem:** `max_tokens=1000` is fixed for all profiles. Visual+Sequential+Active
profiles need structural elements (diagrams, numbered steps, practice prompts) AND
full factual content — impossible in 1,000 tokens. Verbal+Reflective profiles are fine
at 1,000 because their style is plain prose, close to how gold answers are written.

### Step A1 — Add `_get_max_tokens()` method to `TutorAgent`

Find the `TutorAgent` class. Add this method after `__init__()` and before `run_session()`:

```python
def _get_max_tokens(self, reasoning_plan: dict | None) -> int:
    """
    Allocate additional token budget to profiles whose style directives
    require structural formatting (Visual, Sequential, Active).

    These profiles need space for BOTH formatting scaffolding AND full
    factual coverage. A fixed 1,000-token budget forces a tradeoff;
    this method eliminates that tradeoff by scaling the budget to need.

    R0 (reasoning_plan=None) always uses the base budget so the baseline
    is unchanged and the R0/R1 comparison remains controlled.

    Budget scale:
        Base (R0 or Verbal+Reflective+Intuitive+Global): 1,000 tokens
        +200 per structural dimension active:
            Visual    (vis_ver == -1): needs diagrams/tables
            Sequential (seq_glo == -1): needs numbered steps
            Active    (act_ref == -1): needs practice prompts
        Maximum cap: 1,600 tokens
    """
    if reasoning_plan is None:
        return 1000  # R0 baseline — never change this

    vector = reasoning_plan.get("fslsm_vector", {})
    structural_dims = sum([
        vector.get("vis_ver") == -1,    # Visual
        vector.get("seq_glo") == -1,    # Sequential
        vector.get("act_ref") == -1,    # Active
    ])
    return min(1000 + structural_dims * 200, 1600)
```

### Step A2 — Wire `_get_max_tokens()` into `run_session()`

In `run_session()`, find the tutor LLM call:

```python
        # Step 4 — Call tutor LLM
        tutor_resp = self.tutor_client.chat(
            system=system_prompt,
            user=context_prompt,
            max_tokens=self.max_tokens,
        )
```

Replace `max_tokens=self.max_tokens` with the profile-aware call:

```python
        # Step 4 — Call tutor LLM
        tutor_resp = self.tutor_client.chat(
            system=system_prompt,
            user=context_prompt,
            max_tokens=self._get_max_tokens(plan),
        )
```

> `plan` is already defined earlier in `run_session()` as either the reasoning plan
> (R1) or `None` (R0). No other changes to `run_session()` needed.

## Fix B — Reframe style directives as additive wrappers in `profile_agent.py`

**File:** `src/tutor/profile_agent.py`  
**Problem:** Generation directives use imperative framing ("You MUST include an ASCII
diagram") which the LLM interprets as a replacement for prose explanation. Changing
to additive framing ("additionally include") preserves style markers while protecting
factual content. Apply only to the three dimensions that cause structural budget
competition: Visual, Sequential, Active.

### Step B1 — Update `_DIM_GENERATION["vis_ver"][-1]` (Visual)

Find in `_DIM_GENERATION`:

```python
    "vis_ver": {
        -1: (
            "VISUAL LEARNING STYLE — You MUST:\n"
            "- Include at least one ASCII diagram, table, or structured visual\n"
            "- Use markdown tables or formatted layouts to show relationships\n"
            "- Describe spatial relationships and use visual metaphors"
        ),
```

Replace the `-1` value with:

```python
        -1: (
            "VISUAL LEARNING STYLE:\n"
            "- First provide a complete written explanation covering all key facts.\n"
            "- Then ADDITIONALLY include at least one ASCII diagram, markdown table,\n"
            "  or structured visual that illustrates the relationships.\n"
            "- The diagram supplements the explanation — it does not replace written content.\n"
            "- Use visual metaphors and spatial language throughout."
        ),
```

### Step B2 — Update `_DIM_GENERATION["seq_glo"][-1]` (Sequential)

Find:

```python
    "seq_glo": {
        -1: (
            "SEQUENTIAL LEARNING STYLE — You MUST:\n"
            "- Structure your response as numbered steps (Step 1, Step 2, Step 3...)\n"
            "- Present exactly one concept per step before moving to the next\n"
            "- Use transitional phrases ('Now that we understand X, let's move to Y')"
        ),
```

Replace the `-1` value with:

```python
        -1: (
            "SEQUENTIAL LEARNING STYLE:\n"
            "- Structure your response as numbered steps (Step 1, Step 2, Step 3...).\n"
            "- Each step must contain a FULL explanation of that concept — do not\n"
            "  abbreviate factual content to fit the step structure.\n"
            "- Use transitional phrases ('Now that we understand X, let's move to Y').\n"
            "- All key facts from the evidence must appear within the steps."
        ),
```

### Step B3 — Update `_DIM_GENERATION["act_ref"][-1]` (Active)

Find:

```python
    "act_ref": {
        -1: (
            "ACTIVE LEARNING STYLE — You MUST:\n"
            "- Include at least one hands-on exercise or 'Try it yourself' prompt\n"
            "- Use phrases like 'Let's try...', 'Now experiment with...', 'Your turn:'\n"
            "- End with a practice challenge the student can attempt"
        ),
```

Replace the `-1` value with:

```python
        -1: (
            "ACTIVE LEARNING STYLE:\n"
            "- Provide the complete factual explanation first.\n"
            "- At the END of your response, ADDITIONALLY add a short hands-on exercise\n"
            "  or 'Try it yourself' prompt (2–4 sentences maximum).\n"
            "- Use phrases like 'Now try...', 'Your turn:', 'Experiment with...'.\n"
            "- The practice prompt is an appendix to the explanation, not a replacement."
        ),
```

## Fix C — Inject factual anchor into `generate_system_prompt()` in `profile_agent.py`

**File:** `src/tutor/profile_agent.py`  
**Function:** `generate_system_prompt()`  
**Problem:** Even with Fix B reframing the directives, the LLM may still front-load
style structure before laying down factual content. A mandatory STEP 0 anchor forces
a 2–3 sentence factual foundation at the start of every R1 response, guaranteeing the
judge finds core claims even if the rest of the response is heavily formatted.

### Step C1 — Add `_CONTENT_ANCHOR` constant at module level

After the `_DIM_DEPRIORITIZE` block and before `_LABEL_PARTS`, add:

```python
# ---------------------------------------------------------------------------
# Factual anchor injected before style directives in every R1 system prompt.
# Guarantees the judge finds core claims regardless of structural formatting.
# ---------------------------------------------------------------------------
_CONTENT_ANCHOR = (
    "## STEP 0 — FACTUAL FOUNDATION (always do this first)\n"
    "Before applying any style formatting, write 2–3 sentences that directly "
    "answer the question using the key facts from the retrieved evidence. "
    "This factual core MUST appear at the start of your response. "
    "Style formatting is built around this foundation, not instead of it.\n"
)
```

### Step C2 — Prepend anchor in `generate_system_prompt()`

Find `generate_system_prompt()`. Locate where `gen_dir` is composed into the prompt:

```python
        prompt = (
            f"You are an expert AI Tutor for Introductory Machine Learning, "
            f"using the Dive into Deep Learning (D2L) textbook as your knowledge source.\n\n"
            f"The student is a {label} learner. You MUST adapt your response to match "
            f"their learning style. This is NOT optional — your response will be evaluated "
            f"on how well it conforms to the style requirements below.\n\n"
            f"## MANDATORY STYLE REQUIREMENTS\n\n"
            f"{gen_dir}\n"
        )
```

Replace with:

```python
        prompt = (
            f"You are an expert AI Tutor for Introductory Machine Learning, "
            f"using the Dive into Deep Learning (D2L) textbook as your knowledge source.\n\n"
            f"The student is a {label} learner. Your response must achieve two goals "
            f"in strict priority order:\n\n"
            f"{_CONTENT_ANCHOR}\n"
            f"## STYLE REQUIREMENTS (applied after Step 0)\n\n"
            f"{gen_dir}\n"
        )
```
