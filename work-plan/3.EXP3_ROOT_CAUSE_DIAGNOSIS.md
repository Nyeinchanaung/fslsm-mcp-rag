## Root Cause 1 — FAISS Cosine Scores Are Near-Random

### Evidence

```
Optimal in rank 1:     4/30  (13.3%)   ← Only 4 queries correctly retrieved
Optimal in top-5:      7/30  (23.3%)   ← Optimal not in top-5 77% of the time
Average cosine score:  0.15–0.25       ← Expected for relevant content: >0.70
Average score spread:  0.02–0.05       ← Too small for rank-1 to be meaningful
```

### Concrete Examples

```
Query: "How do anchor boxes evolve from labeling to bounding boxes with confidence scores?"
  Rank 1: code_sandbox       score=0.245  (WRONG — code tool for a concept question)
  Rank 2: socratic_dialogue  score=0.229
  Rank 5: worked_example     score=0.200
  Optimal: prose_explainer   NOT IN TOP-5

Query: "How does batch normalization in LeNet influence ResNet architecture?"
  Rank 1: conceptual_overview  score=0.136
  Rank 5: summary_outline      score=0.103
  Score spread: only 0.033 — FAISS is essentially guessing
  Optimal: diagram_renderer    NOT IN TOP-5
```

### Root Cause Explanation

Your queries are **technical D2L content questions** with vocabulary like:
> "ResNeXt", "GoogLeNet", "minibatch SGD", "GloVe loss function", "SNLI dataset"

Your tool descriptions are **generic pedagogical language** with vocabulary like:
> "sequential learners", "hands-on practice", "visual representations"

There is **zero shared vocabulary** between the two. The `text-embedding-3-small` model
places these in completely different regions of the embedding space, producing
cosine similarities of 0.12–0.32 that are effectively random noise.

**Analogy:** Asking FAISS to match "ResNeXt initialization" to "visual learners prefer
diagrams" is like searching for "photosynthesis" to find a menu item.

### Fix

Rewrite all 15 tool descriptions to include **both** generic pedagogy language AND
domain-bridging vocabulary that links pedagogical intent to content patterns:

```python
# BEFORE (too generic — FAISS cannot link to D2L queries)
TOOL_03 = "Break a procedure into ordered steps with observable results.
            Best for sequential thinkers."

# AFTER (bridges pedagogy to content vocabulary)
TOOL_03 = """stepwise_walkthrough: Decompose a complex algorithm, training
             procedure, or architectural evolution into an explicit numbered
             sequence — each step shows one transformation, one computation,
             or one configuration change. Use when the query asks 'what steps',
             'how does X work procedurally', 'trace the execution', or involves
             training loops, forward/backward passes, initialization sequences,
             or layer-by-layer construction. Best for Sequential and Sensing
             learners who need ordered concrete progression through technical
             procedures."""
```

**Target cosine scores after fix:** >0.55 for relevant query-tool pairs.
**Expected TSA improvement:** S1a 6.1% → 25–35%

## Root Cause 2 — Profile Encoding Mismatch in GROUND_TRUTH_MAP

### Evidence

**Actual profile format (from exp2 sessions):**
```json
{
  "act_ref":  -1,   // -1 = Active,      +1 = Reflective
  "sen_int":  -1,   // -1 = Sensing,     +1 = Intuitive
  "vis_ver":  -1,   // -1 = Visual,      +1 = Verbal
  "seq_glo":  -1    // -1 = Sequential,  +1 = Global
}
```

**Confirmed encoding (from profile_label cross-reference):**
```
"Active-Sensing-Visual-Sequential"       → act_ref=-1, sen_int=-1, vis_ver=-1, seq_glo=-1
"Reflective-Intuitive-Visual-Sequential" → act_ref=+1, sen_int=+1, vis_ver=-1, seq_glo=-1
"Reflective-Sensing-Verbal-Global"       → act_ref=+1, sen_int=-1, vis_ver=+1, seq_glo=+1
```

**Confirmed convention:** Negative = first named pole (Active/Sensing/Visual/Sequential)
                         Positive = second named pole (Reflective/Intuitive/Verbal/Global)

### Concrete Labeling Errors Found

```
Profile: act_ref=-1, sen_int=-1, vis_ver=-1, seq_glo=-1
Label:   Active-Sensing-Visual-Sequential
Assigned optimal tool: [11] summary_outline (Reflective+Sequential)
                        ↑ WRONG — this is for a Reflective learner, not Active
Correct optimal tool should be: [1] diagram_renderer (Visual+Sequential)
                              or [3] stepwise_walkthrough (Sequential+Sensing)
```

```
Profile: act_ref=1, sen_int=1, vis_ver=-1, seq_glo=1
Label:   Reflective-Intuitive-Visual-Global
Assigned optimal tool: [14] concept_map (Global+Visual)   ← CORRECT ✓
```

```
Profile: act_ref=1, sen_int=-1, vis_ver=1, seq_glo=-1
Label:   Reflective-Sensing-Verbal-Sequential
Assigned optimal tool: [12] prose_explainer (Verbal+Sequential) ← CORRECT ✓
```

**Estimated error rate:** ~25–30% of optimal labels are incorrectly assigned
due to misinterpretation of the ±1 bipolar encoding.

### Fix

Update the GROUND_TRUTH_MAP logic to correctly decode ±1 profiles:

```python
# scripts/fix_ground_truth_map.py

def decode_profile(fslsm_vector: dict) -> dict:
    """
    Convert ±1 bipolar encoding to named dimension flags.
    
    Input:  {"act_ref": -1, "sen_int": 1, "vis_ver": -1, "seq_glo": 1}
    Output: {"Active":1, "Reflective":0, "Intuitive":1, "Sensing":0,
             "Visual":1, "Verbal":0, "Global":1, "Sequential":0}
    """
    return {
        "Active":      1 if fslsm_vector["act_ref"] < 0 else 0,
        "Reflective":  1 if fslsm_vector["act_ref"] > 0 else 0,
        "Sensing":     1 if fslsm_vector["sen_int"] < 0 else 0,
        "Intuitive":   1 if fslsm_vector["sen_int"] > 0 else 0,
        "Visual":      1 if fslsm_vector["vis_ver"] < 0 else 0,
        "Verbal":      1 if fslsm_vector["vis_ver"] > 0 else 0,
        "Sequential":  1 if fslsm_vector["seq_glo"] < 0 else 0,
        "Global":      1 if fslsm_vector["seq_glo"] > 0 else 0,
    }

# Updated GROUND_TRUTH_MAP using profile_label string (more readable)
PROFILE_TOOL_MAP = {
    # Format: "Dim1-Dim2-Dim3-Dim4" → tool_id
    # Priority: leading dimensions drive tool selection
    
    # Visual dimension leads
    "Active-Sensing-Visual-Sequential":       1,  # diagram_renderer
    "Active-Sensing-Visual-Global":           14, # concept_map
    "Reflective-Intuitive-Visual-Sequential": 1,  # diagram_renderer
    "Reflective-Intuitive-Visual-Global":     14, # concept_map
    "Active-Intuitive-Visual-Sequential":     1,  # diagram_renderer
    "Active-Intuitive-Visual-Global":         2,  # interactive_simulation
    "Reflective-Sensing-Visual-Sequential":   1,  # diagram_renderer
    "Reflective-Sensing-Visual-Global":       14, # concept_map

    # Verbal + Reflective/Sequential
    "Reflective-Sensing-Verbal-Sequential":   12, # prose_explainer
    "Reflective-Sensing-Verbal-Global":       12, # prose_explainer
    "Reflective-Intuitive-Verbal-Sequential": 6,  # abstract_derivation
    "Reflective-Intuitive-Verbal-Global":     6,  # abstract_derivation
    "Active-Sensing-Verbal-Sequential":       3,  # stepwise_walkthrough
    "Active-Sensing-Verbal-Global":           15, # case_study
    "Active-Intuitive-Verbal-Sequential":     6,  # abstract_derivation
    "Active-Intuitive-Verbal-Global":         5,  # conceptual_overview
}

def assign_optimal_tool_v2(fslsm_vector: dict) -> int:
    """Correct optimal tool lookup using profile_label string."""
    dims = decode_profile(fslsm_vector)
    
    act = "Active" if dims["Active"] else "Reflective"
    sns = "Sensing" if dims["Sensing"] else "Intuitive"
    vis = "Visual" if dims["Visual"] else "Verbal"
    seq = "Sequential" if dims["Sequential"] else "Global"
    
    label = f"{act}-{sns}-{vis}-{seq}"
    return PROFILE_TOOL_MAP.get(label, 12)  # fallback: prose_explainer
```

**Expected TSA improvement from fix alone:** +8–12 percentage points

## Root Cause 3 — FSLSM Augmentor Receives Wrong Profile Format

### Evidence

**Augmentor expects (designed format):**
```python
{"act": 1, "ref": 0, "sns": 0, "int": 0, "vis": 1, "vrb": 0, "seq": 0, "glo": 0}
```

**Actual sessions provide:**
```python
{"act_ref": -1, "sen_int": -1, "vis_ver": -1, "seq_glo": -1}
```

**Current augmentor code:**
```python
FIELD_TO_DIM = {
    "act": "Active", "ref": "Reflective",
    "sns": "Sensing", "int": "Intuitive",
    "vis": "Visual",  "vrb": "Verbal",
    "seq": "Sequential", "glo": "Global",
}

def augment(self, query, profile):
    active_dims = [FIELD_TO_DIM[f] for f, v in profile.items()
                   if v == 1 and f in FIELD_TO_DIM]
    # With actual profile {"act_ref":-1,...}: FIELD_TO_DIM["act_ref"] → KeyError
    # So active_dims = [] → NO AUGMENTATION → S1b degrades to S1a
```

**Why S1b still beats S1a (27.5% vs 6.1%):**
If augmentation fails, S1b and S1a should be identical. The gap exists because either:
1. The augmentor was already partially updated to handle ±1 format (inconsistently)
2. The S1b code path uses the `profile_label` string for augmentation separately

**Check:** Add this logging to verify:
```python
aug = augmentor.augment(query, session["fslsm_vector"])
print(f"Aug length: {len(aug)} chars (>len(query) = augmentation worked)")
```

### Fix

Update the augmentor to handle ±1 bipolar encoding:

```python
# fslsm_query_augmentor.py — updated augment() method

def augment(self, query: str, profile: dict) -> str:
    """
    Handles BOTH profile formats:
      Old binary:  {"act":1, "ref":0, "vis":1, ...}
      New bipolar: {"act_ref":-1, "vis_ver":-1, ...}
    """
    # Detect format
    if "act_ref" in profile:
        # New ±1 bipolar format — decode to named dims
        active_dims = []
        if profile["act_ref"] < 0:   active_dims.append("Active")
        else:                         active_dims.append("Reflective")
        if profile["sen_int"] < 0:   active_dims.append("Sensing")
        else:                         active_dims.append("Intuitive")
        if profile["vis_ver"] < 0:   active_dims.append("Visual")
        else:                         active_dims.append("Verbal")
        if profile["seq_glo"] < 0:   active_dims.append("Sequential")
        else:                         active_dims.append("Global")
    else:
        # Old binary format
        active_dims = [
            FIELD_TO_DIM[f] for f, v in profile.items()
            if v == 1 and f in FIELD_TO_DIM
        ]

    if not active_dims:
        return query

    directives = "; ".join(DIM_DIRECTIVES[dim] for dim in active_dims
                           if dim in DIM_DIRECTIVES)
    return f"{query}; {directives}"
```

## Root Cause 4 — S1a Structural TSA Ceiling (Design Constraint)

### Evidence

Same query appears with multiple profiles and different optimal tools:
```
Query: "How do vanishing gradients affect optimization algorithms?"
  Profile Active-Sensing-Verbal-Global   → optimal: concept_map   (14)
  Profile Active-Sensing-Verbal-Verbal   → optimal: prose_explainer (12)
  Profile Reflective-Sensing-Verbal-Seq  → optimal: prose_explainer (12)
```

```
Query: "Trace the evolution of neural network architectures..."
  Profile Reflective-Sensing-Visual-Global  → optimal: abstract_derivation (6)
  Profile Reflective-Sensing-Verbal-Global  → optimal: interactive_simulation (2)
```

### Explanation

S1a retrieves based on **raw query only** (no profile). For the same query, S1a always
selects the same tool. But the ground truth is **profile-dependent** — the same query
has different optimal tools for different student profiles.

**Mathematical ceiling for S1a:**
If each query appears for avg. N distinct profiles with distinct optimal tools,
S1a can match at most 1/N of those sessions correctly.

For your 5,760 sessions with 16 profiles and 50 questions each:
→ Each query appears up to 16 times with up to 7–8 distinct optimal tools
→ S1a TSA ceiling ≈ 1/7 ≈ **14.3%**

**S1a's actual 6.1% is below even this ceiling**, which confirms Root Causes 1 and 2
are also corrupting S1a results (wrong FAISS retrieval + wrong labels).

### This Is By Design

S1a is supposed to underperform because it ignores profiles — that's the ablation's point.
The S1b vs S1a gap (21.4 pp) is intended to quantify FSLSM conditioning's value.
After fixing Root Causes 1–3, S1a TSA should rise to its natural ceiling (~14%), and
S1b should reach 45–55% by correctly conditioning on profiles.

## prose_explainer Dominance: A False Positive

### Evidence from TSA by optimal tool (50 sessions):
```
[12] prose_explainer        12/16 hits (75%)   ← Dominant
[2]  interactive_simulation  3/6  hits (50%)
[15] case_study              0/6  hits (0%)
[3]  stepwise_walkthrough    0/5  hits (0%)
[6]  abstract_derivation     0/4  hits (0%)
[1]  diagram_renderer        0/4  hits (0%)
```

### Explanation

`prose_explainer` is both the most commonly **assigned optimal tool** (20.66%)
AND the most commonly **selected by FAISS** because its description contains
words like "explanation", "describe", "verbal" that appear in many D2L query
patterns. This creates a correlated true-positive: FAISS happens to select it,
and it happens to be correct, inflating TSA artificially.

For all other tools (diagram_renderer, stepwise_walkthrough, etc.), FAISS
never selects them when they are the optimal tool → 0% TSA.

**This means your 27.5% S1b TSA is almost entirely driven by prose_explainer
chance matching — not by genuine FSLSM-conditioned tool selection.**
